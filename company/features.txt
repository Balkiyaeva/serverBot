Название компании: АО "НИТ" - Акционерное общество "Национальные информационные технологии".
Для автоматизации бизнес процессов этой компании было разработано веб-приложение для легкого управления данными о реестре государственных услуг.
В Реестре Государственных Услуг (наименование веб приложения) - далее "РГУ", доступны следующие роли:
 1. Администратор
 2. Аналитик

 Администратор назначается непосредственно через базу данных. Его назначает сопровождающий проект разработчики - Амира и Елхан.
 Аналитик может быть добавлен:
 1. либо посредством оставления заявки через регистрацию, которая находится по следующей ссылке: localhost:3000/register. После заявку должен одобрить один из администраторов в своей панели управления.
 2. либо посредством прямого добавления администратором в панели управления. Администратор должен ввести почту, роль и проект аналитика.

 У Аналитика, как и у Администратора может быть один из следующих проектов: "ПЭП", "МГОВ", "ЕЛ".

 Основная цель этого веб-приложения автоматизировать управление данными и минимизировать ручной ввод.
 Ежегодно данные должны быть актуализированы.
 Чтобы предотвратить ручной ввод, приложение предоставляет возможность загрузить таблицу, которая будет считывать данные и хранить их базе данных, отображая их на веб страничке.
 Чтобы так же следить за всеми изменениями, можно переключаться между таблицами: Основная таблица, Добавленные новые данные, Данные отсутствующие в новой таблице.
 в Основной таблице хранятся как и прошлые данные, так и новые загруженные данные.
 в Добавленных новых данных, соответственно, хранятся новые загруженные данные.
 в Данные отсутствующие в новой таблице хранятся данные, которые ранее были в основной таблице, но их не было в новой загруженной таблице. Эти данные в идеале должны быть удалены, но это остается за Администратором / Аналитиком.

 Аналитик имеет следующие возможности:
 1. Просмотр таблицы (localhost:3000/table)
 2. Смена таблиц (Основная таблица, Добавленные новые данные, Данные отсутствующие в новой таблице) (находятся в нижнем правом углу)
 3. Поиск по всей таблице (слева, вверху таблицы)
 4. Фильтрация столбцов как в Excel (над каждым столбцом кнопка фильтрации, открывает модальное окно)
 5. Редактирование и/или удаление строк (крайние левые кнопки в каждой строке). Однако, для того, чтобы удалить, пользователь должен сначала отметить строку (checkbox), а потом нажать на кнопку мусорного ведра справа вверху таблицы.
 6. Создание новых строк (справа вверху таблицы, первая кнопка).
 7. Загрузка новой таблицы в веб приложение (справа вверху таблицы, крайняя кнопка)
 8. Скачать на компьютер таблицу (справа вверху таблицы, третья кнопка)
 9. Просматривать журнал изменений
 10. Удалять старые данные из журнала изменений
 11. Выйти из веб-приложения (пользователь должен нажать на свое имя, а потом из выплывающих строк выбрать опцию "Выйти")

 Администратор может все то, что может аналитик, а так же:
 1. Просмотр панели управления (самая верхняя кнопка справа посередине)
 2. Добавление нового пользователя / принимать или отклонять заявки на регистрацию от пользователей / редактировать данные пользователей
 Для того, чтобы добавить нового пользователя, администратор должен войти в панель управления. Сверху будет форма для добавления нового пользователя. Он должен ввести его почту, роль (пока доступна только опция Аналитик) и проект
 Для того, чтобы принять/отклонить заявку нового пользователя, администратор должен зайти в панель управления. в таблице пользователей подсвеченными будут отображаться те, кто ожидает подтверждения.
 3. Редактирование и/или удаление столбцов
 4. Создание новых столбцов

 Все изменения, а именно: редактирование, удаление, добавление, загрузка новой таблицы отображаются в журнале изменений, находящийся слева на главной странице.
 Чтобы открыть ее, пользователь должен найти кнопку '>' в левом верхнем углу.
 Все изменения отслеживаются в реальном времени, и пользователь может сразу замечать их.
